# 프로젝트 점검 사이트 (베타 테스트)

## 무엇을 만들었나요?

사내 프로젝트 진행 상황을 **순서도(플로우차트)** 형태로 관리하는 웹 도구입니다.
로그인/회원가입 없이 링크만 있으면 바로 접속해서 사용할 수 있습니다.

기존에는 "진행 상황을 표/리스트로만 관리"하는 방식을 생각했는데, 실제 업무는 단계가 갈라지기도 하고
(부품 A 검토 / 부품 B 검토를 동시에 진행) 다시 합쳐지기도 해서, 그 흐름을 그대로 그림으로 그리면서
각 단계에 메모를 남기고 처리 여부를 체크할 수 있는 도구로 방향을 잡았습니다.

## 주요 기능

- **프로젝트 생성**: 이름만 입력하면 시작 → 중간 단계 → 완료, 기본 3칸짜리 순서도가 자동으로 생성됩니다.
- **프로젝트 이름 변경**: 프로젝트 목록에서 연필 아이콘을 누르면 이름을 바꿀 수 있습니다.
- **칸(노드) 추가/삭제**: 필요한 만큼 중간 단계를 늘리거나 줄일 수 있습니다. 칸에 마우스를 올리면 상/하/좌/우 4방향에 **+ 아이콘**이 나타나고, 이미 다른 칸과 연결된 방향은 자동으로 숨겨집니다.
- **분기(branch) / 합류(merge)**: 한 단계에서 여러 갈래로 나뉘었다가, 나중에 다시 하나로 합쳐지는 흐름도 표현할 수 있습니다. 갈래마다 단계 수가 달라도 자동으로 높이를 맞춰 정렬됩니다.
- **완료 표시(체크박스)**: 각 칸 좌상단의 체크박스로 진행 완료 여부를 표시할 수 있습니다. 완료로 체크하면 업무 내용 글자 색이 회색으로 바뀝니다.
- **칸 위치 직접 옮기기**: 자동 배치된 칸을 마우스로 드래그해서 원하는 위치로 옮길 수 있고, 새로고침해도 옮긴 위치가 그대로 유지됩니다.
- **연결선만 선택해서 삭제**: 칸 사이의 화살표(연결선)를 클릭하면 노란색으로 표시되고, 그 상태에서 Delete 키를 누르면 양쪽 칸은 그대로 두고 연결만 끊을 수 있습니다.
- **업무 내용 + 메모**: 칸을 클릭하면 옆에 패널이 열리고, 그 단계에서 할 일과 메모를 남길 수 있습니다.
- **처리 여부 체크 + 답글**: 메모를 처리했으면 체크 표시가 되고, 어떻게 처리했는지 답글(대댓글)로 기록을 남길 수 있습니다. 처리된 메모는 칸 위 아이콘 모양이 바뀌어서 한눈에 구분됩니다.
- **저장 버튼**: 사실 모든 변경사항은 입력하는 즉시 자동으로 저장되지만, 눈으로 확인하고 싶을 때를 위해 "저장" 버튼을 누르면 "저장됨"으로 잠깐 표시됩니다.
- **클라우드 저장 (여러 명이 같은 프로젝트 사용 가능)**: 모든 데이터는 이제 공용 서버(Supabase)에 저장됩니다. 사내 누구든 링크에 접속하면 같은 프로젝트 목록과 순서도를 보고 수정할 수 있습니다.
- **다른 사람이 보고 있는 프로젝트 표시**: 첫 화면 프로젝트 목록에서, 지금 누군가 열어서 보고 있는 프로젝트 옆에는 "편집 중" 표시가 뜹니다.
- **변경사항 알림**: 내가 보고 있는 프로젝트를 다른 사람이 수정하면 화면 상단에 "다른 사용자가 수정했습니다 · 새로고침" 배너가 뜹니다. 클릭하면 최신 내용을 다시 불러옵니다 (자동으로 화면이 바뀌지는 않습니다 — 작업 중 갑자기 화면이 바뀌는 걸 막기 위함).
- **내보내기 / 가져오기**: 프로젝트를 JSON 파일로 저장했다가 나중에 다시 불러올 수 있습니다 (백업 및 다른 사람에게 파일로 전달 가능).
- **실행 취소 / 다시 실행 (Ctrl+Z / Ctrl+Shift+Z, Ctrl+Y)**: 칸/연결선 추가·삭제, 이동, 완료 체크, 제목/메모 저장 등을 되돌리고 다시 실행할 수 있습니다.
- **여러 칸 복사/붙여넣기 (Ctrl+C / Ctrl+V)**: Shift+클릭으로 여러 칸을 선택한 뒤 복사하면, 선택한 칸들과 그 사이 연결선까지 통째로 복제됩니다 (시작/완료 칸은 복사 대상에서 자동 제외).
- **가로/세로 방향 전환**: 상단 "저장" 버튼 왼쪽의 방향 전환 버튼으로 순서도를 위→아래(세로) 또는 왼쪽→오른쪽(가로)으로 바꿀 수 있습니다. 프로젝트마다 따로 저장되고, 누가 보든 같은 방향으로 보입니다.
- **다크 테마**의 깔끔한 화면.

## 사용법

1. 첫 화면에서 **"새 프로젝트"** 클릭 → 프로젝트 이름 입력 (예: `F29H859 프로세서 모듈`, `신제품 개발` 등) → 생성
2. 시작 / (빈) / 완료, 3칸짜리 순서도가 자동으로 만들어집니다.
3. 칸에 **마우스를 올리면** 상/하/좌/우에 작은 **+ 아이콘**이 나타납니다. 클릭한 방향으로 새 칸이 뻗어나가며 추가됩니다 (예: 오른쪽 + 클릭 시 오른쪽으로 새 칸 생성). 이미 그 방향으로 연결된 적이 있으면 해당 아이콘은 표시되지 않습니다.
4. 칸 위/아래의 **⊙ 아이콘**(이중 동그라미)을 드래그해서
   - **빈 공간에 놓으면** 새 칸(분기)이 추가됩니다.
   - **다른 칸 위에 놓으면** 그 칸과 합류(merge) 연결됩니다.
5. 칸은 **마우스로 드래그**해서 원하는 위치로 자유롭게 옮길 수 있습니다.
6. 칸을 **클릭**하면 오른쪽에 상세 패널이 열리고, 업무 내용을 적거나 메모를 남길 수 있습니다.
7. 칸 좌상단의 **체크박스**를 누르면 그 단계가 완료 표시되고 업무 내용 글자가 회색으로 바뀝니다.
8. 메모의 체크박스를 누르면 "처리 완료"로 표시되고, 처리한 내용을 답글로 남길 수 있습니다.
9. 선택된 칸은 **노란 테두리**로 표시됩니다. 선택한 상태에서 **Delete 키**를 누르면 그 칸을 삭제할 수 있습니다 (시작/완료 칸은 삭제 불가).
10. 칸 사이의 **연결선을 클릭**하면 노란색으로 선택되고, **Delete 키**를 누르면 그 연결만 삭제됩니다 (칸은 남아있음).
11. 상단의 **"저장"** 버튼을 누르면 지금 상태가 저장되었다는 표시("저장됨")가 잠깐 나타납니다 (참고: 실제로는 변경할 때마다 이미 자동 저장되고 있어서 안 눌러도 안전합니다).
12. 상단의 **"내보내기"**로 지금 프로젝트를 파일로 저장할 수 있고, 첫 화면의 **"가져오기"**로 그 파일을 다시 불러올 수 있습니다.
13. 프로젝트 목록에서 각 항목 오른쪽 연필 아이콘으로 이름을 바꿀 수 있고, 휴지통 아이콘으로 삭제할 수 있습니다.
14. 실수했을 때는 **Ctrl+Z**로 되돌리고, **Ctrl+Shift+Z**(또는 **Ctrl+Y**)로 다시 실행할 수 있습니다.
15. **Shift+클릭**으로 여러 칸을 선택한 뒤 **Ctrl+C**로 복사, **Ctrl+V**로 붙여넣으면 선택했던 칸들(과 그 사이 연결선)이 통째로 복제되어 나타납니다. 복제된 칸은 다른 칸과 연결되어 있지 않으니 드래그해서 원하는 곳에 연결하면 됩니다.
16. 첫 화면 프로젝트 목록에서 초록색 **"편집 중"** 표시가 있으면 지금 다른 사람이 그 프로젝트를 보고 있다는 뜻입니다.
17. 프로젝트를 보는 중에 상단에 **"다른 사용자가 수정했습니다 · 새로고침"** 배너가 뜨면, 다른 사람이 이 프로젝트를 방금 고쳤다는 뜻입니다. 클릭하면 최신 내용을 불러옵니다.
18. 상단 "저장" 버튼 왼쪽의 **가로/세로 버튼**을 누르면 순서도 방향이 바뀌면서 자동으로 다시 배치됩니다 (손으로 옮겨둔 칸 위치는 초기화됩니다). 방향을 바꾸고 나서 당황했다면 **Ctrl+Z**를 누르세요 — 방향과 손으로 옮겼던 위치까지 방금 전 상태 그대로 정확히 되돌아갑니다.

## 지금 단계에서 꼭 알아두실 점

- 데이터는 이제 **공용 서버(Supabase)에 저장**됩니다. 링크에 접속하는 사내 누구나 로그인 없이 모든 프로젝트를 보고 수정할 수 있습니다 (사내용 도구이므로 별도 권한 구분은 없습니다).
- 같은 프로젝트를 동시에 여러 명이 열어볼 수 있지만, **다른 사람의 수정 내용이 내 화면에 실시간으로 자동 반영되지는 않습니다.** 대신 상단에 알림 배너가 뜨고, "새로고침"을 눌러야 최신 내용을 받아옵니다. 정확히 같은 칸을 동시에 고치면 나중에 저장한 쪽 내용이 남습니다.
- 지금은 **"이런 도구 어떤가요?" 사용성 피드백을 받는 단계**입니다.
- 저장/불러오기가 실패하면 화면에 오류 내용을 보여주는 창이 뜹니다. 창에 있는 **"복사"** 버튼으로 오류
  내용을 복사해서 알려주시면 원인을 빠르게 확인할 수 있습니다.
