# 사내 프로젝트 현황 공유 사이트

## 목적

사내 개발 프로젝트의 진행상황을 점검하고 아이디어를 공유하기 위한 웹페이지.
로그인 없이, 비공개 주소를 아는 사내 인원만 접속하여 사용.

## 사용 방식

- 여러 명이 동시에 접속해서 실시간으로 글을 남기고, 항목을 추가/체크할 수 있어야 함
- 로그인/회원가입 없음 (누구인지 구분할 필요는 없거나, 이름만 입력하는 정도로 충분)
- 검색엔진 노출 차단 (robots.txt 등으로 비공개 유지)

## 담고 싶은 데이터 (초안 — 필요시 수정)

- 프로젝트명
- 진행상황 / 상태 (예: 진행중, 보류, 완료 등)
- 담당자
- 체크 여부 (완료 체크박스)
- 아이디어 게시판: 제목, 내용, 작성자, 작성일

## 기술 구성

- 프론트엔드: 정적 웹페이지 (HTML/CSS/JS 또는 React)
- 백엔드/DB: Supabase (실시간 동기화 지원, 무료 티어)
- 배포: Vercel (또는 Netlify), GitHub 저장소(private)와 연동
- 사용자 환경: 사용자는 C 언어는 알지만 웹 개발 경험 없음. 웹 관련 개념(비동기 처리, 환경변수 등)은 필요할 때마다 설명 부탁드림

## 진행 절차

1. Supabase 프로젝트 생성 및 테이블 설계 (데이터 구조 확정)
2. 프론트엔드 화면 구현 (진행상황 리스트 + 아이디어 게시판 + 체크박스 UI)
3. Supabase 연동 코드 작성 (데이터 읽기/쓰기, 실시간 반영 여부 결정)
4. GitHub private 저장소 생성 및 코드 push
5. Vercel 프로젝트 생성 및 배포, 환경변수(API 키 등)는 Vercel 환경변수 설정에 등록
6. robots.txt 추가 및 URL 비공개 유지 (검색엔진 미노출)
7. 배포 확인 후 사내 공유

## 참고

- 로그인 없는 비공개 주소 방식은 보안이 아니라 URL 추측 방지에 의존하는 방식임 — 민감 정보(고객 데이터 등)는 올리지 않음
- 규모는 개인 사이드 프로젝트 수준 (코드량 수백 줄, 완성까지 하루~주말 정도 예상)
- Supabase, GitHub, Vercel 가입 및 API 키 발급 등 계정/설정 관련 작업은 사용자가 직접 진행해야 하며, 클로드 코드가 단계별로 안내

## 요청 사항

위 내용을 바탕으로 Supabase 테이블 스키마 설계부터 시작하여, 프론트엔드 구현과 배포까지 단계별로 진행해줘. 각 단계에서 사용자가 직접 해야 할 작업(계정 가입, 값 복사 등)이 있으면 명확히 안내할 것.
